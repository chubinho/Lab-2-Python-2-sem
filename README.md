# Лабораторная работа №2. Модель задачи: дескрипторы и @property

## Описание работы
Второй этап разработки платформы посвящен созданию защищенной доменной модели задачи `Task`. Основной фокус сделан на управлении доступом к атрибутам, валидации данных и защите инвариантов состояния объекта.

В работе реализовано разделение между публичным API и внутренним состоянием с использованием протокола дескрипторов, что позволяет предотвратить некорректные состояния объекта еще на этапе присваивания значений.

## Функционал
* **Валидация через Data Descriptors**: Автоматическая проверка типов и бизнес-логики для ID, приоритета и статуса задачи.
* **Контроль жизненного цикла**: Использование Read-Only дескрипторов для фиксации времени создания задачи и предотвращения его изменения.
* **Вычисляемые свойства**: Применение `@property` для динамического определения готовности задачи к выполнению.
* **Разграничение приоритетов**: Демонстрация работы Data и Non-Data дескрипторов в цепочке поиска атрибутов Python.
